# MiniKV v1.0.0

MiniKV v1.0.0 is a bounded educational append-only key-value engine for studying
durability boundaries, fail-closed recovery, atomic replacement, portable
backup, and cooperative single-writer coordination.

The package remains classified as beta because its verified local POSIX scope
does not support a broader production-readiness claim.

## Included

- Typed dependency-free `open`, `get`, `put`, `delete`, `keys`, `stats`,
  `compact`, `backup`, `restore`, and `close` API.
- Versioned CRC-protected log with explicit limits and torn-final-frame recovery.
- Canonical atomic compaction and SHA-256 validated backup/restore.
- POSIX lifetime lock with independent-process crash-release evidence.
- Privacy-safe operational events and deterministic synthetic workload lineage.
- Reproducible offline demo and verified wheel/sdist release policy.

## Verification

The release candidate is tested on CPython 3.11, 3.12, and 3.13 on GitHub's
Ubuntu runners. CI runs Ruff, strict MyPy, 93 tests, demo regeneration, workload
evidence, package build and inspection, isolated wheel installation, dependency
consistency, and vulnerability audit.

## Honest scope

MiniKV is educational local storage, not a replacement for SQLite, RocksDB,
Redis, or a production database. It does not provide Windows support,
distributed locking, multi-writer transactions, encryption, authenticated
provenance, remote retention, or physical power-loss certification.
